**Model Identification Code** (идентификатор модели) — это самый первый элемент в коде модуля данных (DMC). Если продолжать нашу аналогию с адресом, то это **название города** или **индекс проекта**.

Простыми словами: этот код говорит системе, к какому именно изделию (вертолету, танку, станку) относится вся последующая информация.

---

### 1. Зачем он нужен?
В рамках одной компании могут выпускаться десятки модификаций одного и того же изделия. Например, вертолет Ми-171 имеет множество версий: гражданские, военные, арктические.
Чтобы документация для модели **A3** случайно не попала в руководство для модели **E**, каждому проекту присваивают свой уникальный буквенно-цифровой код.

### 2. Структура кода
Обычно он состоит из 2–14 символов (букв и цифр). В вашем примере `MI171A3`:
* **MI171** — база или семейство изделия (Ми-171).
* **A3** — конкретная модификация или серия.



### 3. На что он влияет?
Model Ident Code — это «фильтр верхнего уровня». Когда техник открывает электронный справочник (IETP), он первым делом выбирает модель. Как только выбрана `MI171A3`, система скрывает все инструкции, которые относятся к другим версиям вертолета, чтобы механик не применил деталь или метод ремонта от старой модели к новой.

### 4. Важные правила
* **Уникальность:** В рамках одной организации (или отрасли) этот код не должен повторяться.
* **Регистрация:** Эти коды обычно жестко прописываются в «Бизнес-правилах» (Business Rules) проекта в самом начале разработки.
* **Статичность:** В отличие от кода системы, этот параметр практически никогда не меняется в процессе жизни документации для конкретной модели.

---

### Резюме (собираем пазл целиком)
Теперь у вас есть полная картина того, как S1000D находит нужную страницу:

1.  **Model Ident (`MI171A3`):** О каком изделии говорим? — *О вертолете Ми-171А3.*
2.  **System Code (`D00`):** О какой части? — *О двигателе.*
3.  **Info Code (`520`):** Что делаем? — *Устанавливаем.*
4.  **Variant (`A`):** Каким способом? — *Стандартным.*

Получается уникальный «отпечаток пальца» для каждого кусочка информации.
