В спецификации **S1000D** параметр **System Code** (системный код) — это «адрес» конкретной системы внутри сложного изделия (самолета, корабля или автомобиля).

Если говорить совсем просто: представьте, что всё ваше изделие — это огромный многоквартирный дом. Тогда **System Code** — это номер подъезда или обозначение конкретной инженерной сети (например, «Отопление» или «Лифты»).

Вот основные моменты, которые помогут разложить это по полочкам:

### 1. Зачем он нужен?
В современных технических руководствах информация разбита на маленькие блоки (Data Modules). Чтобы компьютер и техник понимали, к чему относится конкретная инструкция, используется **SNS** (Standard Numbering System — стандартная система нумерации). System Code — это первая и самая главная часть этой системы.

### 2. Что значат эти буквы и цифры (например, D00)?
Код обычно состоит из трех символов. Он жестко иерархичен:

* **Первый символ (D):** Обозначает крупный класс систем. Например, в авиации это может быть «Двигатель» или «Фюзеляж».
* **Второй и третий символы (00):** Уточняют конкретную подсистему.
    * Если стоит `00`, значит, это **общая информация** по всей системе (общие правила безопасности, описание работы системы в целом).
    * Если цифры меняются (например, `D10`, `D20`), значит, речь идет об узких узлах внутри этой системы.

### 3. Пример для наглядности
Допустим, мы описываем велосипед по логике S1000D:

| Код системы | Что это означает |
| :--- | :--- |
| **A00** | **Рама** (общая информация) |
| **B00** | **Тормозная система** (в целом) |
| **B10** | Передний тормоз |
| **B20** | Задний тормоз |
| **C00** | **Трансмиссия** (цепи, педали) |

### Резюме
**System Code** — это фундамент структуры данных. Он позволяет быстро отфильтровать тысячи документов и оставить только те, что касаются, например, только «системы кондиционирования» (код которой может быть зафиксирован как `D00` в конкретном проекте).

Без этого кода в S1000D наступил бы хаос, так как невозможно было бы автоматически собрать руководство для конкретной части машины.
