# Руководство пользователя "Gazprom MI Render"

## 1. Введение

"Gazprom MI Render" — это настольное приложение, разработанное для создания, просмотра и управления технической документацией в формате S1000D. Оно позволяет работать с отдельными модулями данных (XML-файлами), импортировать контент из формата Markdown и автоматически генерировать сводное оглавление проекта (PMC).

## 2. Начало работы

При первом запуске приложения вы увидите экран с одной кнопкой — "Открыть проект". Вся работа в приложении ведется в контексте проектной папки.

### 2.1. Создание нового проекта

1.  Создайте на вашем компьютере пустую папку, в которой будет храниться проект.
2.  В приложении нажмите кнопку "Открыть проект".
3.  В появившемся системном диалоге выберите созданную вами папку.
4.  Поскольку это новый проект, приложение откроет окно "Настройки нового проекта". Заполните поля необходимыми данными. Эти настройки будут использоваться для автоматической генерации метаданных в XML-файлах.
5.  В полях есть кнопка "i", нажав на нее вы получите пояснение, зачем это поле и что оно делает
6.  Нажмите "Сохранить". В папке проекта будет создан файл `project.json` с вашими настройками, и проект будет открыт.

### 2.2. Открытие существующего проекта

1.  Нажмите "Открыть проект" (либо на стартовом экране, либо через иконку папки в правом верхнем углу, если другой проект уже открыт).
2.  Выберите папку существующего проекта (папку, в которой уже есть файл `project.json`).
3.  Приложение загрузит настройки из `project.json` и откроет проект.

## 3. Основные функции

После открытия проекта на главном экране станут доступны следующие действия:

### 3.1. Оглавление проекта (PMC)

*   **Открыть оглавление:** Эта кнопка ищет в папке проекта файл оглавления (`PMC-*.XML`). Если файл найден, он открывается для просмотра. Если нет — он генерируется автоматически на основе всех `DMC-*.XML` файлов в проекте и затем открывается.
*   **Иконка в меню (Перестроить оглавление):** Кнопка с иконкой списка в правом верхнем углу всегда принудительно пересоздает файл оглавления, даже если он уже существует. Используйте её, если вы добавили, удалили или изменили файлы в проекте вручную.

### 3.2. Создание нового XML-файла (чек-лист)

1.  Нажмите кнопку "Создать XML файл (чек-лист)".
2.  В появившемся диалоге введите метаданные для нового модуля данных:
    *   **Info Name:** Человекочитаемое название (например, "Ежедневный осмотр").
    *   **System Code, Info Code, Variant:** Коды, уникально идентифицирующие модуль данных в соответствии со стандартом S1000D. Приложение автоматически предлагает следующий доступный `Info Code`.
5. В полях есть кнопка "i", нажав на нее вы получите пояснение, зачем это поле и что оно делает
6. Поле Info Code подсвечивается цветом выбранного типа процедуры (Аварийная, Обычная)
4. Нажмите "Создать".
6. Приложение сгенерирует S1000D-совместимый XML-файл с базовой структурой для чек-листа (`crew.xsd`), сохранит его в папку проекта и откроет в режиме редактирования.

### 3.3. Импорт из Markdown

Эта функция позволяет быстро преобразовать текстовые файлы (`.md` или `.txt`) в структурированные S1000D XML. Вверху справа есть кнопка "Помощь" где в открывшихся разделах вы сможете увидеть пример заполнения шаблона

1.  Нажмите одну из кнопок импорта:
    *   **Импорт чеклиста из MD:** для создания чек-листов и процедур (`crew.xsd`).
    *   **Импорт описания из MD:** для создания описательных модулей (`descript.xsd`).
2.  Выберите `.md` или `.txt` файл для импорта.
3.  Заполните метаданные для нового XML-файла в диалоговом окне (аналогично созданию нового файла). Название по умолчанию берется из первого заголовка в Markdown-файле.
4.  Нажмите "Импорт".
5.  Приложение сконвертирует ваш файл в XML, сохранит его в папку проекта и откроет для просмотра/редактирования.

### 3.4. Открытие произвольного XML-файла

Кнопка "Открыть XML файл" позволяет выбрать и просмотреть любой XML-файл с вашего компьютера, не обязательно относящийся к текущему проекту.

### 3.5. Редактирование документа и его сохранение

При открытии любого XML-документа (будь то чек-лист, процедура или описание) он открывается в режиме **просмотра**. Для внесения изменений необходимо переключиться в режим **редактирования**:

1. В правом верхнем углу окна открытого документа найдите переключатель **"Редактировать"**.
2. Включите его (переключатель станет зеленым). Интерфейс документа изменится:
   - Текст превратится в редактируемые поля ввода.
   - Появятся кнопки добавления новых элементов (шагов, примечаний, предупреждений, заголовков).
   - Рядом с существующими элементами появятся иконки удаления (в виде красной корзины).
3. Вносите необходимые изменения: редактируйте текст, добавляйте новые шаги (например, вызов/ответ для чек-листа), назначайте роли экипажа (кнопка `+` рядом с шагом).
4. Если вы внесли хотя бы одно изменение, рядом с переключателем режимов появится **красная иконка дискеты** (кнопка сохранения), предупреждающая о несохраненных данных.
5. Нажмите на иконку дискеты, чтобы сохранить изменения. После успешного сохранения иконка станет серой, а внизу экрана появится зеленое уведомление "Изменения успешно сохранены!".

**Важно:** Если вы попытаетесь закрыть документ с несохраненными изменениями (нажав кнопку "Назад"), приложение покажет диалоговое окно с предупреждением, чтобы предотвратить случайную потерю данных.

#### 3.5.1. Редактирование параметров документа

Если вам необходимо изменить коды документа (например, переместить его из категории "Нормальная эксплуатация" в "Аварийные ситуации" или поменять название), это можно сделать прямо в окне просмотра/редактирования:

1. Откройте документ (чек-лист или описание).
2. В правом верхнем углу (рядом с переключателем режима) нажмите на **иконку шестеренки (Настройки файла)**.
3. Откроется диалоговое окно с параметрами файла (System Code, Info Code, Variant, Название).
4. Внесите нужные изменения. Приложение автоматически проверит, не заняты ли новые коды другим файлом.
5. Нажмите "Сохранить".
6. Приложение автоматически переименует физический файл на диске в соответствии с новыми кодами, обновит XML-структуру и "в фоне" пересоберет файл оглавления проекта (PMC), чтобы он сразу отобразился в нужном разделе.

### 3.6. Создание файла оглавления

Модуль публикации (PMC, Publication Module) выполняет роль оглавления проекта. Он собирает все ваши разрозненные файлы модулей данных (DMC) в единую структуру (книгу).

В приложении есть два способа взаимодействия с оглавлением:

**Способ 1: Просмотр текущего оглавления**
На главном экране нажмите кнопку **"Открыть оглавление"**.
- Если файл `PMC-*.XML` уже существует в папке проекта, приложение просто откроет его для просмотра.
- Если файла еще нет, приложение автоматически просканирует папку проекта, найдет все `DMC-` файлы, сгенерирует новый файл `PMC-` и откроет его.

**Способ 2: Принудительное обновление оглавления**
Если вы добавили новые файлы, удалили старые или изменили их названия, ваше текущее оглавление устареет.
- На главном экране, в правом верхнем углу (рядом с иконкой папки), нажмите на **иконку списка (Перестроить оглавление)**.
- Приложение заново просканирует всю папку проекта и перезапишет файл `PMC-*.XML` с актуальными данными, после чего откроет его.

Файл оглавления строится автоматически на основе метаданных, сохраненных в `project.json` (информация о компании, модели, языке и т.д.).

### 3.7. Редактирование параметров проекта

Если после создания проекта вам потребовалось изменить его базовые настройки (например, название модели, реквизиты компании, коды языка или данные BREX), вы можете сделать это на специальном экране:

1. Откройте проект.
2. В правом верхнем углу главного экрана (в верхней панели AppBar) найдите и нажмите **иконку шестеренки (Настройки проекта)**.
3. Откроется экран со всеми параметрами, которые вы заполняли при создании проекта.
4. Внесите необходимые изменения (например, исправьте опечатку в названии компании).
5. Нажмите иконку **дискеты (Сохранить)** в правом верхнем углу.
6. Приложение выполнит массовое обновление:
   * Сохранит новые параметры в `project.json`.
   * Откроет все XML-файлы (`DMC-*.XML`) в проекте и перезапишет в них старые метаданные на новые (копирайты, языки, коды и т.д.).
   * Перестроит файл оглавления (PMC).

### 3.8. Валидация XML документов (S1000D)

В приложении встроен строгий валидатор, который позволяет мгновенно проверить, соответствует ли ваш документ стандарту S1000D (версии 4.1). Валидация работает локально и не требует подключения к интернету.

1. Откройте любой XML-документ (чек-лист или описание).
2. В правом верхнем углу (в верхней панели AppBar) найдите **иконку щита с галочкой** (кнопка "Проверить S1000D").
3. Нажмите на нее. Приложение в фоновом режиме проанализирует структуру вашего файла на основе официальных XSD-схем S1000D.
4. Вы мгновенно получите результат в виде всплывающего окна:
   * **✓ Валиден (зеленый):** Документ полностью соответствует международному стандарту.
   * **✗ Ошибки валидации (красный):** В документе есть нарушения. Приложение укажет точные номера строк и суть ошибки (например: ожидался тег `para`, но найден `title`, или отсутствуют обязательные элементы S1000D).
   
Это позволяет гарантировать, что все созданные вами файлы будут без проблем загружены в любую стороннюю CSDB (Common Source Database).

## 4. Файловая структура проекта

*   `project.json`: Файл с настройками вашего проекта. Содержит информацию о модели, компании, языковые коды и т.д.
*   `DMC-*.XML`: Файлы модулей данных (Data Modules). Это основные рабочие файлы, содержащие контент (описания, процедуры, чек-листы).
*   `PMC-*.XML`: Файл модуля публикации (Publication Module). Является сводным оглавлением, которое ссылается на все DMC-файлы в проекте. Генерируется автоматически.

## 5. Работа с Оглавлением

Окно оглавления (PM Viewer) работает как встроенный файловый менеджер вашего проекта. 

* **Режимы отображения:** По умолчанию оглавление открывается в виде красивой сетки из "плиток" (папок). Каждая плитка соответствует определенной категории (например, "040: Аварийные ситуации" или "060: Нормальная эксплуатация"). В правом верхнем углу есть кнопка переключения вида (Плитки / Сплошной список), если вы предпочитаете классическое текстовое представление дерева.
* **Цветовая индикация типов документов:** Плитки категорий окрашиваются в разные цвета для быстрого визуального распознавания, что критично для авиационной документации:
  * Красный — Аварийные ситуации (040)
  * Оранжевый — Сложные ситуации (050)
  * Зеленый — Контрольные карты (030)
  * Синий — Описания систем / КСЭИС (041)
* **Дерево файлов (Drill-down):** При нажатии на любую плитку категории вы "проваливаетесь" внутрь раздела. Заголовок экрана меняется на название категории, а на экране отображается список конкретных XML-документов (шагов и процедур), относящихся к этому разделу. Нажав кнопку "Назад" в левом верхнем углу (или системным жестом "назад"), вы вернетесь обратно к списку плиток-категорий.
